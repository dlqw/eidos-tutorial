# Eidos 0.6.0-alpha.1 dot Namespace syntax

- Reserved `::` exclusively for declaration binding and changed all package, module, type, trait, effect, constructor, compile-time-value, and member qualification to `.`.
- Standardized ADT constructor lists on comma separators; `|` is rejected by the Eidos 0.6 parser and handled only by explicit migration from Eidos 0.5.
- Documented the uppercase compile-time/lowercase runtime naming tiers, `lower_snake_case` runtime style, and lowercase dependency alias plus uppercase module rule.
- Migrated every tutorial example, standard-library reference, FFI example, BNF production, and project manifest to Eidos 0.6.0-alpha.1.
